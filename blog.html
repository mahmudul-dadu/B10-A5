What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. With the DOM, programmers can access and manipulate HTML or XML documents as if they were using an object-oriented programming language.

How do you select an element from the DOM?

To select an element from the DOM (Document Object Model) in JavaScript, you can use several methods depending on the type of selection you need. Here are the most common methods:

1. Select by id: Use document.getElementById("id") to select a single element by its id attribute.

2. Select by class: Use document.getElementsByClassName("class-name") to select all elements with a specific class name. This returns an HTMLCollection (a list-like array of elements).

3. Select by tag: Use document.getElementsByTagName("tag-name") to select all elements with a specific tag (e.g., div, p, button). This also returns an HTMLCollection.

4. Select with querySelector (CSS-style selector): Use document.querySelector("selector") to select the first element that matches a CSS selector.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation refers to the process of using event propagation (bubbling) to handle events at a higher level in the DOM than the element on which the event originated. It allows us to attach a single event listener for elements that exist now or in the future.

Understanding the DOM as an Object Hierarchy

Its good to think of the DOM as an object made up of objects. All of the elements in your HTML become an element object. Much like an array object, this allows you to call useful methods on it. When the browser parses the HTML and creates the DOM, it structures everything like a tree. Every set of tags becomes a branch with as many sub branches as needed. Every branch, or node, has a parent. All nodes have parents apart from one, the root node, which is called document.
